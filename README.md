# NotificationSystem

Микросервисная система уведомлений на .NET с единым API и подключаемыми каналами доставки (Email / SMS). Репозиторий содержит отдельные проекты для API, каналов уведомлений и общий модуль контрактов.

**Цель сервиса:** принимать запросы на отправку уведомлений, маршрутизировать их по каналу и обеспечивать единый контракт для интеграции с другими сервисами, гарантируя надежность доставки через асинхронную обработку.

## Состав репозитория

- **Notification.Gateway** — HTTP API Gateway (точка входа для других сервисов)
- **Notification.EmailService** — канал доставки Email (фоновый сервис)
- **Notification.SmsService** — канал доставки SMS (фоновый сервис)
- **Notification.Contracts** — общие контракты, DTO и модели, используемые всеми модулями

## Возможности

- **Единый REST API** для отправки уведомлений через различные каналы
- **Асинхронная обработка** через RabbitMQ (гарантированная доставка, повторные попытки)
- **Масштабируемая архитектура** (каждый канал — независимый микросервис)
- **Мониторинг статусов** — REST API для проверки состояния отправленных уведомлений
- **Подготовлено для контейнеризации** (Dockerfile для каждого сервиса)
- **Автоматическое восстановление** при сбоях (retry-логика, dead letter queues)

## Что используется

- **RabbitMQ** — брокер сообщений для асинхронной коммуникации между сервисами
- **PostgreSQL** — единая база данных для хранения статусов всех уведомлений
- **.NET 8** — основная платформа разработки
- **Entity Framework Core** — ORM для работы с базой данных
- **Docker & Docker Compose** — контейнеризация и оркестрация сервисов
